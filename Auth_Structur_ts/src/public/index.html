Layered architecture • JWT dual-token

Route → Controller → Service → Model — strict separation of concerns, centralized error handling, and secure authentication. Dark mode + responsive UX.

Route + DTO

Endpoint definitions & validation middleware. Malformed requests never reach business logic, drastically protecting system computing overhead and security layers.

Controller

Orchestrator between requests/responses & services. Extracts payload targets parameters, utilizing global try/catch block schemes to bubble exceptions to the core handler cleanly.

Service Layer

Core business logic: DB queries via Mongoose, automated JWT generations, token refreshing arrays, and email communication integrations. Completely decoupled and framework-agnostic.

Data Access (Models)

Handles exact DB architecture boundaries via schema mappings. Enforces pre-save structural lifecycle hooks to run critical encryption hashing models directly inside the safe storage level.

Security Implementation

Uses short-lived validation Access Tokens along with durable Refresh Tokens safely locked in isolated HttpOnly, Secure, and SameSite=Strict cookie structures to completely negate XSS and CSRF strategies.

Reliability & Standardized Structure

Decoupled exceptions utilize a specialized apiError processing constructor. Successful communications conform directly to a predictive structural format contract:

{
  "success": true,
  "message": "Operation successful",
  "data": { ... }
}
